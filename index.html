📌 GitHub Issues를 데이터베이스로 활용

각 제출물이 하나의 Issue로 저장됩니다.

  1. Repository Issues에 제출물 저장
  2. 라벨로 분류 (완료/미완료)

📝 GitHub Gist로 간편 저장

각 제출물이 개별 Gist로 저장됩니다.

  • 계정 없이도 익명 제출 가능
  • 고유 URL 생성
  • 버전 관리 자동

📁 Repository 직접 저장

지정된 Repository에 파일로 저장됩니다.

  • 폴더 구조로 체계적 관리
  • 일괄 다운로드 가능
  • GitHub Actions 연동 가능

💡 Token 생성 방법:

  1. GitHub → Settings → Developer settings
  2. Personal access tokens → Generate new token
  3. 권한: repo, gist 체크
  4. 생성된 토큰 복사 (한 번만 표시됨)
